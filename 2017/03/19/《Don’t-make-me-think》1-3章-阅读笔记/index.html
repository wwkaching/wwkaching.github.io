《Don’t make me think》1-3章 阅读笔记

可用性的定义

  • 有用：是否帮助人们完成一些必需的事物？
  • 可学习：人们能否明白如何使用它？
  • 可记忆：人们每次使用的时候，是否都需要重新学习？
  • 有效：它们能完成任务吗？
  • 高效：它们是否需要花费适当的时间和努力就能完成任务？
  • 合乎期望：是人们想要的吗？
  • 令人愉悦：人们使用的时候觉得有意思甚至好玩吗？

一、指导原则

别让我思考

当看到一个页面时，它应该是不言而喻、一目了然、自我解释的，用户应该能明白——它是什么意思，怎么样用它，而不需要进行额外的思考。

用户在网站不应该多花时间思考的事：

  • 我在什么位置？
  • 我该从哪里开始？
  • 他们把XX放在什么地方了？
  • 这个页面上最重要的是什么？
  • 为什么他们把它叫这个？
  • 这是广告还是网站的内容？

如果不能让一个网站做到不言而喻，那么至少应该让它自我解释。

二、我们实际上是如何使用Web的

  • 我们不是阅读，而是扫描。
  • 我们不作最佳选择，而是满意即可。
  • 我们不是追根究底，而是勉强应付。

三、广告牌设计101法则

如果用户都是疾驰而过，需要注意：

1.尽量利用习惯用法

适当的遵循习惯和惯例，能让用户更容易理解，如果不打算使用一种已有的Web习惯用法，必须确认你在使用一种：

  • 同样清楚，同样不言而喻，没有学习曲线的一种方法——这样就跟习惯用法一样好了。
  • 带来很大的价值，因此值得用户付出一点努力来学习的方法。

2.建立有效的视觉层级

一个视觉层级清楚的页面有三个特点：

  • 越重要的部分越突出
  • 逻辑上相关的部分也在视觉上相关
  • 逻辑上包含的部分在视觉上进行嵌套

3.把页面划分为明确定义的区域

这可以让用户很快决定关注页面的哪些区域，或者放心的跳过哪些区域。

4.明显标识可点击的地方

通过视觉线索，例如形状、位置、格式区分能否点击。

5.降低视觉噪音

有三类噪音：

  • 眼花缭乱：页面上所有的内容都希望得到注意，那么效果可能适得其反。
  • 组织不当：页面上的东西扔得乱七八糟
  • 太过密集：内容太多，找不到真正重要得。

当设计页面时，这样做可能是一个好方法：先假定所有得内容都是视觉噪音，并去除任何对页面没有帮助得内容。

6.为文本设置格式，以便扫描

  • 充分使用标题
  • 保持段落简洁
  • 使用符号列表
  • 突出关键词语
